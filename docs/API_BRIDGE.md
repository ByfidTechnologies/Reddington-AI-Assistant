# Reddington API Bridge – C# ↔ Python Communication\n\n## Overview\n\nThe bridge layer enables seamless communication between the C# desktop application and the Python FastAPI backend using **gRPC** with **REST API** fallback.\n\n## Communication Protocol\n\n### Primary: gRPC\n\n**Advantages:**\n- High performance (HTTP/2)\n- Protocol buffers serialization\n- Bidirectional streaming\n- Type-safe\n- Low latency\n\n### Fallback: REST API\n\nIf gRPC fails, the system automatically falls back to REST endpoints.\n\n## Message Formats\n\n### User Input Request\n\n```json\n{\n  \"type\": \"text|voice\",\n  \"content\": \"User input text or speech\",\n  \"timestamp\": 1717395631000,\n  \"session_id\": \"uuid-here\",\n  \"context\": {\n    \"user_name\": \"John\",\n    \"location\": \"Home\",\n    \"last_intent\": \"weather\"\n  }\n}\n```\n\n### AI Response\n\n```json\n{\n  \"status\": \"success|error\",\n  \"response_text\": \"AI response here\",\n  \"audio_base64\": \"base64-encoded-audio\",\n  \"action\": \"open_app|control_device|show_info|play_audio\",\n  \"action_payload\": {\n    \"app\": \"calculator\",\n    \"params\": {}\n  },\n  \"timestamp\": 1717395632000\n}\n```\n\n### System Status Request\n\n```json\n{\n  \"request_type\": \"system_status\"\n}\n```\n\n### System Status Response\n\n```json\n{\n  \"backend_online\": true,\n  \"version\": \"1.0.0\",\n  \"services\": {\n    \"chatgpt\": {\"status\": \"connected\", \"last_check\": 1717395600000},\n    \"google\": {\"status\": \"connected\", \"last_check\": 1717395600000},\n    \"alexa\": {\"status\": \"connected\", \"last_check\": 1717395600000},\n    \"weather\": {\"status\": \"connected\", \"last_check\": 1717395600000}\n  },\n  \"permissions\": {\n    \"microphone\": true,\n    \"camera\": true,\n    \"admin\": true\n  },\n  \"internet\": {\n    \"connected\": true,\n    \"download_mbps\": 85.3,\n    \"upload_mbps\": 42.1\n  }\n}\n```\n\n## REST API Endpoints\n\n### Core Endpoints\n\n```\nGET  /health                    - Health check\nGET  /api/status                - Backend status\nPOST /api/process-input         - Process user input\nGET  /api/permissions           - Check permissions\nPOST /api/initialize            - Initialize backend\nGET  /api/system-status         - Full system status\n```\n\n### Audio Endpoints\n\n```\nWS   /ws/audio-stream           - WebSocket for real-time audio\nPOST /api/audio/transcribe      - STT (Speech-to-Text)\nPOST /api/audio/synthesize      - TTS (Text-to-Speech)\n```\n\n### Service Endpoints\n\n```\nGET  /api/services/status       - All services status\nPOST /api/services/refresh      - Refresh service connections\nGET  /api/services/{service}    - Specific service status\n```\n\n### AI Endpoints\n\n```\nPOST /api/ai/query              - Query AI\nPOST /api/ai/context            - Update AI context\nGET  /api/ai/models             - Available models\n```\n\n### Smart Home Endpoints\n\n```\nGET  /api/smarthome/devices     - List devices\nPOST /api/smarthome/control     - Control device\nGET  /api/smarthome/status      - Device status\n```\n\n## WebSocket Protocol\n\n### Audio Streaming\n\n**Connection:**\n```\nWS ws://localhost:5000/ws/audio-stream\n```\n\n**Client Message (Audio Data):**\n```json\n{\n  \"type\": \"audio\",\n  \"data\": \"base64-encoded-audio-chunk\",\n  \"format\": \"wav|pcm\",\n  \"sample_rate\": 16000\n}\n```\n\n**Server Response (Streaming):**\n```json\n{\n  \"type\": \"processing\",\n  \"status\": \"recognizing\",\n  \"interim_text\": \"partial transcription\"\n}\n```\n\n**Final Response:**\n```json\n{\n  \"type\": \"complete\",\n  \"text\": \"full transcription\",\n  \"confidence\": 0.95,\n  \"response\": {\n    \"text\": \"AI response\",\n    \"audio\": \"base64-audio\"\n  }\n}\n```\n\n## Error Handling\n\n### Error Response Format\n\n```json\n{\n  \"status\": \"error\",\n  \"error_code\": \"SERVICE_UNAVAILABLE|INVALID_INPUT|PERMISSION_DENIED\",\n  \"message\": \"Human-readable error message\",\n  \"details\": {\n    \"service\": \"chatgpt\",\n    \"action\": \"query\",\n    \"timestamp\": 1717395632000\n  }\n}\n```\n\n### Common Error Codes\n\n| Code | Meaning |\n|------|----------|\n| `SERVICE_UNAVAILABLE` | Backend service offline |\n| `INVALID_INPUT` | Invalid request format |\n| `PERMISSION_DENIED` | Insufficient permissions |\n| `TIMEOUT` | Request timeout |\n| `INTERNAL_ERROR` | Internal server error |\n\n## C# Client Example\n\n```csharp\npublic class BridgeClient\n{\n    private readonly HttpClient _httpClient;\n    private const string BackendUrl = \"http://localhost:5000\";\n    \n    public async Task<AIResponse> ProcessInputAsync(string input)\n    {\n        var request = new UserInputRequest\n        {\n            Type = \"text\",\n            Content = input,\n            Timestamp = DateTimeOffset.UtcNow.ToUnixTimeMilliseconds()\n        };\n        \n        var response = await _httpClient.PostAsJsonAsync(\n            $\"{BackendUrl}/api/process-input\", request);\n        \n        return await response.Content.ReadAsAsync<AIResponse>();\n    }\n}\n```\n\n## Python Server Example\n\n```python\nfrom fastapi import FastAPI, WebSocket\nfrom typing import Dict, Any\n\napp = FastAPI()\n\n@app.post(\"/api/process-input\")\nasync def process_input(request: Dict[str, Any]):\n    user_input = request.get(\"content\")\n    intent = await ai_engine.recognize_intent(user_input)\n    response = await ai_engine.generate_response(intent)\n    return {\n        \"status\": \"success\",\n        \"response_text\": response[\"text\"],\n        \"action\": response[\"action\"]\n    }\n\n@app.websocket(\"/ws/audio-stream\")\nasync def websocket_endpoint(websocket: WebSocket):\n    await websocket.accept()\n    async for data in websocket.iter_json():\n        # Process audio\n        result = await process_audio(data)\n        await websocket.send_json(result)\n```\n\n## Connection Lifecycle\n\n```\n1. C# App Starts\n   ↓\n2. Initialize Bridge Connection\n   ↓\n3. Send /api/initialize\n   ↓\n4. Python Backend Responds\n   ↓\n5. Periodic Health Checks (every 30s)\n   ↓\n6. Process User Input\n   ↓\n7. Receive Response\n   ↓\n8. Update UI\n```\n\n## Reconnection Strategy\n\nIf connection fails:\n\n1. **Attempt 1-3**: Retry immediately\n2. **Attempt 4-6**: Wait 1 second, then retry\n3. **Attempt 7-10**: Wait 5 seconds, then retry\n4. **Attempt 11+**: Wait 30 seconds, then retry (max 5 times)\n5. **Final**: Show error to user, allow manual reconnect\n\n## Security\n\n- **HTTPS only** in production\n- **API Key authentication** (header: `X-API-Key`)\n- **CORS validation** (localhost only by default)\n- **Input validation** on both sides\n- **Rate limiting** (100 requests per minute per IP)\n\n## Performance Targets\n\n- **Response time**: < 500ms for simple queries\n- **Audio latency**: < 1 second end-to-end\n- **Connection latency**: < 100ms\n- **Throughput**: 1000+ requests/second\n\n---\n\nFor implementation details, see code examples in the repository.\n
