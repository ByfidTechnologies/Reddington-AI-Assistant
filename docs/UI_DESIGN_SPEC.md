# Reddington UI Design Specification\n\n## Design Philosophy\n\nReddington's interface represents a **futuristic vision of modern AI**, combining:\n- **High-tech aesthetics** with minimal complexity\n- **Fluid animations** that feel responsive and alive\n- **3D elements** that enhance spatial understanding\n- **Elegant simplicity** despite technical sophistication\n\n## Color Palette\n\n### Primary Colors\n\n| Name | Hex | RGB | Usage |\n|------|-----|-----|-------|\n| **Deep Purple** | `#2D1B4E` | 45, 27, 78 | Primary backgrounds, dominant UI |\n| **Navy Blue** | `#0F1B4E` | 15, 27, 78 | Secondary backgrounds, accents |\n| **Midnight Black** | `#000000` | 0, 0, 0 | Text, dark elements |\n| **Accent Gold** | `#D4AF37` | 212, 175, 55 | Highlights, interactive elements |\n| **Pure White** | `#FFFFFF` | 255, 255, 255 | Text, bright accents |\n\n### Secondary Colors\n\n| Name | Hex | Usage |\n|------|-----|-------|\n| **Status Green** | `#00D084` | Success, online status |\n| **Status Yellow** | `#FFC107` | Warning, pending status |\n| **Status Red** | `#FF3B30` | Error, offline status |\n| **Dark Gray** | `#1A1A1A` | Subtle backgrounds |\n| **Light Gray** | `#E8E8E8` | Borders, dividers |\n\n## Typography\n\n### Font Family\n- **Primary**: Segoe UI (Windows), San Francisco (macOS fallback)\n- **Monospace**: Consolas for code/technical text\n\n### Font Sizes\n\n```\nH1 (Title):     32px, Bold, Letter-spacing: 0.5px\nH2 (Subtitle):  24px, Semibold, Letter-spacing: 0.3px\nH3 (Heading):   18px, Semibold\nBody (Default): 14px, Regular\nSmall (Label):  12px, Regular\nCaption:        10px, Regular, Uppercase\n```\n\n### Font Weights\n- **Regular**: 400\n- **Semibold**: 600\n- **Bold**: 700\n\n## Component Library\n\n### 1. Voice Visualizer\n\n**Description**: Animated audio waveform that responds to user voice\n\n**States**:\n- **Idle**: Subtle pulsing animation, dim opacity (40%)\n- **Listening**: Active waveform following audio input\n- **Processing**: Loading animation with rotating elements\n- **Speaking**: Synchronized with TTS output\n\n**Animation**: \n- Smooth sine wave motion\n- 60 FPS rendering\n- Color transitions between states\n\n```xaml\n<local:VoiceVisualizer\n    x:Name=\"VoiceViz\"\n    State=\"Listening\"\n    Color=\"#D4AF37\"\n    Amplitude=\"0.8\"/>\n```\n\n### 2. Status Indicator\n\n**Description**: Circular indicator showing permission/service status\n\n**Colors**:\n- рҹҹў Green (#00D084): Connected, Available\n- рҹҹЎ Yellow (#FFC107): Pending, Checking\n- рҹ”ҙ Red (#FF3B30): Offline, Error\n\n**Animation**: Smooth color transition, 200ms\n\n### 3. Chat Bubble\n\n**User Message**:\n- Background: Deep Purple (#2D1B4E)\n- Text: White\n- Alignment: Right\n- Border radius: 12px\n\n**AI Message**:\n- Background: Navy Blue (#0F1B4E)\n- Text: White\n- Alignment: Left\n- Border radius: 12px\n- Optional avatar on left\n\n**Animation**: Slide in from bottom, 300ms ease-out\n\n### 4. Button Styles\n\n**Primary Button**:\n```\nBackground: Gold (#D4AF37)\nText: Black (#000000)\nHover: Brightness +10%\nActive: Brightness -10%\nBorder-radius: 8px\nPadding: 12px 24px\nFont-weight: Semibold\nTransition: All 200ms\n```\n\n**Secondary Button**:\n```\nBackground: Transparent\nBorder: 2px Gold (#D4AF37)\nText: Gold\nHover: Background Gold (20% opacity)\nBorder-radius: 8px\nPadding: 10px 20px\n```\n\n### 5. Input Fields\n\n**Style**:\n```\nBackground: Dark Gray (#1A1A1A)\nBorder: 1px Light Gray (#E8E8E8)\nBorder-focus: 2px Gold (#D4AF37)\nText-color: White\nPadding: 12px 16px\nBorder-radius: 6px\nFont-size: 14px\n```\n\n**Placeholder**:\n```\nColor: Gray (50% opacity)\nFont-style: Normal\n```\n\n### 6. Loading Animation\n\n**Concept**: Futuristic circular loader\n\n**Animation**:\n- Rotating circle with gradient\n- Color: Deep Purple вҶ’ Gold вҶ’ Deep Purple\n- Duration: 2 seconds\n- Easing: Linear infinite\n\n### 7. 3D Environment View\n\n**Description**: 3D visualization of user's room/environment\n\n**Features**:\n- Interactive rotation (mouse drag)\n- Zoom controls (mouse wheel)\n- Real-time object detection overlay\n- Temperature sensor display\n- Camera feed integration\n\n**Technology**: Unity or Babylon.js renderer\n\n### 8. Timings\n\n```\nQuick (snappy):        150ms - 200ms\nNormal (standard):     300ms - 400ms\nSlow (dramatic):       500ms - 700ms\nExtra slow (intro):    1000ms - 2000ms\n```\n\n### Easing Functions\n\n```\nQuick In:    EaseInQuad (0.5s)\nQuick Out:   EaseOutQuad (0.3s)\nSoft In/Out: EaseInOutCubic (0.4s)\nBounce:      EaseOutBounce (0.6s)\nSpring:      Custom spring physics\n```\n\n### Page Transitions\n\n- **Fade**: 200ms, opacity 0 вҶ’ 1\n- **Slide**: 300ms, transform translateX\n- **Scale**: 250ms, scale 0.95 вҶ’ 1.0\n- **Rotate**: 400ms, rotateZ 5deg вҶ’ 0\n\n## Branding\n\n###
