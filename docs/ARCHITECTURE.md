# Reddington Architecture – Technische Spezifikation\n\n## System-Übersicht\n\nReddington ist eine **Hybrid-Architektur** mit klarer Separation of Concerns:\n\n### Layer-Struktur\n\n1. **C# (.NET) – Desktop Application & System Integration**\n   - WinUI 3 GUI mit futuristischem Design\n   - Systemintegration (Windows APIs, Registry, Autostart)\n   - Berechtigungsverwaltung\n   - Python-Prozess-Verwaltung\n   - gRPC/REST Bridge zu Python\n\n2. **Python – KI Backend & Automatisierung**\n   - FastAPI Server\n   - KI-Integration (ChatGPT, Claude, Gemini)\n   - Sprachverarbeitung (STT/TTS)\n   - Smart Home Integration\n   - Datenanalyse & Vision Processing\n   - Cloud Service Integration\n\n3. **Communication Bridge**\n   - gRPC für Performance\n   - REST API fallback\n   - WebSocket für Echtzeit-Audio\n   - JSON-RPC für Events\n\n## C# Komponenten\n\n### Core\n- **ApplicationCore.cs** – Hauptanwendungs-Controller\n- **PermissionManager.cs** – Berechtigungsverwaltung\n- **SystemIntegration.cs** – Windows API Integration\n- **ProcessManager.cs** – Python Backend-Verwaltung\n\n### UI (WinUI 3 / MVVM)\n- **MainWindow.xaml** – Haupt-Interface\n- **BootProtocolPage.xaml** – Start-Protokoll\n- **DashboardPage.xaml** – Hauptdashboard\n- **ChatPage.xaml** – Chat-Interface\n- **SettingsPage.xaml** – Einstellungen\n\n### Services\n- **BridgeService.cs** – gRPC/REST Kommunikation\n- **AudioService.cs** – Mikrofon & Speaker Management\n- **HardwareService.cs** – Kamera, Sensoren, Hardware\n- **StorageService.cs** – Lokale & Cloud Storage\n\n### Protocols\n- **FirstStartProtocol.cs** – Erststart-Einrichtung\n- **BootProtocol.cs** – Start-Sequenz\n- **PermissionProtocol.cs** – Permission-Handling\n\n## Python Backend\n\n### FastAPI Server (main.py)\n- Listening on localhost:5000\n- CORS configured for C# app\n- Health check endpoint\n- RESTful API endpoints\n- WebSocket support für Audio\n\n### Core Services\n- **ai_engine.py** – KI-Integration (ChatGPT, Claude, Gemini)\n- **speech_processor.py** – STT/TTS/NLP\n- **intent_resolver.py** – User Intent Analysis\n- **decision_engine.py** – Service Selection\n\n### External Services\n- **smart_home/** – Alexa, Tapo, Reolink\n- **cloud/** – Google Drive, Photos, Maps\n- **communication/** – WhatsApp, Gmail, Messenger\n- **analysis/** – Behavior Analysis, Reports\n\n### Vision Engine\n- **environment_scanner.py** – 3D Room Scanning\n- **object_detection.py** – Computer Vision (YOLO, OpenCV)\n- **spatial_mapper.py** – 3D Space Mapping\n\n## System Workflows\n\n### First Start Protocol\n```\n1. Installation Complete\n   ↓\n2. Admin Rights Check\n   ↓\n3. Permission Request (Microphone, Camera, Filesystem, etc.)\n   ↓\n4. Internet & Service Connectivity Check\n   ↓\n5. Service Authentication (ChatGPT, Google, Alexa, etc.)\n   ↓\n6. Personalization Questions\n   ↓\n7. 3D Environment Scanning\n   ↓\n8. Setup Complete → Boot Protocol\n```\n\n### Boot Protocol (Every Startup)\n```\n1. Loading Animation\n   ↓\n2. Permission Status Check (Red/Yellow/Green)\n   ↓\n3. Internet Speedtest\n   ↓\n4. Service Health Check\n   ↓\n5. Fetch New Messages\n   ↓\n6. Get Weather & Environment Info\n   ↓\n7. Load Main Dashboard\n   ↓\n8. Ready for User Input\n```\n\n### User Input Processing\n```\n1. User Input (Voice or Text)\n   ↓\n2. STT if Voice\n   ↓\n3. Intent Recognition (NLP)\n   ↓\n4. Decision Engine (Which Service?)\n   ↓\n5. Execute Service\n   - ChatGPT/Claude/Gemini for AI queries\n   - Google Services for search/maps\n   - Smart Home for device control\n   - Communication for messages\n   ↓\n6. Format Response\n   ↓\n7. TTS + UI Update\n   ↓\n8. Present to User\n```\n\n## Permission Model\n\n| Permission | Type | Purpose | Status |\n|-----------|------|---------|--------|\n| Administrator | 🔴 Required | System Integration | CRITICAL |\n| Microphone | 🟡 Optional | Voice Recognition | Enhanced |\n| Camera | 🟡 Optional | Video Calls, Environment Scan | Enhanced |\n| Filesystem | 🟢 Limited | App Data Access | Standard |\n| Network | 🔴 Required | API Communication | CRITICAL |\n| Registry | 🔴 Admin-only | System Configuration | CRITICAL |\n| GPU | 🟡 Optional | AI Acceleration | Optional |\n| Printer | 🟡 Optional | Document Printing | Optional |\n\n## Error Handling\n\n- **Local Logging** – All errors logged locally in `/logs`\n- **No PII** – Personally Identifiable Information never logged\n- **User Notification** – Errors displayed to user\n- **Auto-Recovery** – System attempts automatic recovery\n- **Graceful Degradation** – Missing services don't crash system\n\n## Security\n\n- **Encryption** – Sensitive data encrypted locally (AES-256)\n- **API Keys** – Stored in .env, never in code\n- **SSL/TLS** – All external APIs use HTTPS\n- **Local Execution** – AI models can run locally\n- **No Cloud Storage** – User data stays on device by default\n\n## Performance Optimizations\n\n- **Async/Await** – Non-blocking operations\n- **Connection Pooling** – Reuse API connections\n- **Caching** – In-memory caching of frequent queries\n- **Multi-threading** – Separate threads for UI, Audio, Bridge\n- **Resource Management** – Monitor and limit memory usage\n\n## Testing Strategy\n\n### Unit Tests (C#)\n- PermissionManager tests\n- AudioService tests\n- BridgeService tests\n\n### Integration Tests (Python)\n- AI Engine tests\n- Speech Processor tests\n- Service Integration tests\n\n### E2E Tests\n- Full workflow tests\n- Permission flow tests\n- Boot protocol tests\n\n---\n\nFor implementation details, see individual component documentation.\n
